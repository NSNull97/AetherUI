## Required Migration Shape

- `AetherAppearanceStyle` is fixed to `.iOS26` and `.iOS27`; no automatic style, custom style IDs, OS resolving, or fallback chain.
- `AetherApplication` owns a single app-level style through `AppearanceStyle(.iOS26/.iOS27)` and `.appearanceStyle(...)`.
- `AetherAppEnvironmentValues` carries both style and resolved `AetherAppearance` for app and scene render contexts.
- Visible bars update from the current appearance by invalidating/updating their existing renderer views, not by recreating windows or root controllers.
- Local screen-specific customization moves to partial `AetherAppearanceOverride` values and surface resolvers, not to app-wide theme replacement.
- Legacy theme APIs remain only as deprecated compatibility adapters to prevent breaking existing call sites during migration.
